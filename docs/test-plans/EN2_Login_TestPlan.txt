# eNOW Login Feature Test Plan

## Requirements Analysis

| Scenario Name | Requirement | Test Case |
|---------------|-------------|-----------|
| Scenario 1: Login to eNOW | 1.1 The Login screen should display when users click on the eNOW link | 1.1.1 Verify the Login screen appears when clicking the eNOW link |
|  | 1.2 The Login screen should have a configurable image on the right half | 1.2.1 Verify right half of screen presents a configurable image |
|  | 1.3 The Login screen should have a language dropdown in the top left | 1.3.1 Verify top left provides a language dropdown with all available languages |
|  | 1.4 The Login screen should have correct labels and text entry elements | 1.4.1 Verify screen label shows "Login" |
|  |  | 1.4.2 Verify a label for the text entry box shows "Email" |
|  |  | 1.4.3 Verify a text entry box with default contents "Enter email" |
|  | 1.5 The Login screen should have a Next button with configurable color enabled by default | 1.5.1 Verify a "Next" button with configurable color is present and enabled by default |
|
| Scenario 2: Click on "Language" Dropdown | 2.1 Language dropdown should display list of available languages | 2.1.1 Verify clicking on language dropdown presents list of available languages |
|  | 2.2 Available languages should include English and Spanish | 2.2.1 Verify available languages include English |
|  |  | 2.2.2 Verify available languages include Spanish |
|
| Scenario 3: Click on a Language from the "Language" Dropdown | 3.1 Selected language should become the setting for use within eNOW | 3.1.1 Verify the language selected becomes the setting for use within eNOW |
|  | 3.2 Login page contents should be presented in the selected language | 3.2.1 Verify contents on Login page are presented in the language selected |
|  | 3.3 Language dropdown should be dismissed after selection | 3.3.1 Verify the language dropdown is dismissed after selection |
|
| Scenario 4: User Clicks on the Next Button - No Value Entered in Text Box | 4.1 System should display error message when Next is clicked with no email entered | 4.1.1 Verify message "Enter an email" is displayed when Next button is clicked with no value in text box |
|
| Scenario 5: User Enters Text into Email Address Text Box | 5.1 System should display validation message when user starts typing | 5.1.1 Verify message "Please enter a valid email address" is displayed when user starts typing |
|  | 5.2 Validation message should stay until a valid email is entered | 5.2.1 Verify validation message remains displayed until a valid email is entered |
|
| Scenario 6: User Enters a Valid Email Address | 6.1 System should recognize valid email format | 6.1.1 Verify email format <letters/numbers/./->@<letters and numbers>.<letters/numbers/.> is accepted as valid |
|  | 6.2 System should stop displaying validation message when valid email is entered | 6.2.1 Verify validation message is dismissed when valid email format is entered |
|
| Scenario 7: User Clicks on the Next Button - Value Entered in Text Box | 7.1 System should validate whether the email is registered with eNOW | 7.1.1 Verify system validates whether email is registered with eNOW when Next button is clicked with valid email |
|
| Scenario 8: Email is Not Registered with eNOW | 8.1 System should display error message for unregistered email | 8.1.1 Verify message "Couldn't find an account for the email address provided." is displayed when email is not registered |
|
| Scenario 9: Email is Registered with eNOW | 9.1 System should present password entry mechanism when email is registered | 9.1.1 Verify password entry mechanism is presented when email is confirmed to be registered |
